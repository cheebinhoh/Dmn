Publish an item to all registered subscribers. The publish() call is non-blocking (with blocking optional) with respect to subscriber handling: it schedules the delivery in the publisher's asynchronous thread context (so the caller returns quickly) while delivery to each subscriber is dispatched into the corresponding subscriber's asynchronous context.

Parameters
itemThe data item to publish.
blockThe caller will be blocked waiting for item to be published, default is false.

Core implementation that performs buffering and iterates subscribers to dispatch notifications. This runs inside the publisher's asynchronous context (scheduled by publish())

Subclasses may override to customize behavior, but must honor the locking and delivery expectations used by register/unregister and destructor.

Register a subscriber of class interface/subclass from Dmn_Sub with this publisher. After registration, items in the publisher's buffer are replayed to the subscriber.
